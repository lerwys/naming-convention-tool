Validation

Device names are validated against the following set of rules in the device registry:
  1. Instance index (Idx) shall be alphanumeric. I.e., only upper and lower case alphanumeric characters (a-z, A-Z, 0-9) are allowed.
  2. The device names Sec-Sub : Dis-Dev-Idx shall be distinguishable, which means that names shall be unique irrespective of
    • Letter case
    • Letters I, l and number 1
    • Letter O and number 0
    • Letters V and W
    • Leading zeros, i.e., number 0 immediately following a non-numerical character
  3. The maximum length of instance index (Idx) is 6 characters.
Proposed name elements in the area structures are validated against the following set of rules:
  1. Names elements for any of the items in the area structure shall be alphanumeric. I.e., only upper and lower case alphanumeric characters (a-z, A-Z, 0-9) are allowed.
  2. Name elements for super section (Sup) and section (Sec) shall be globally distinguishable.
  3. Name elements for subsection Sub shall be distinguishable under the same parent section.
  4. Names elements for any of the items in the area structure shall have 1 to 6 charachers.
Proposed name elements in the Device structure are validated against the following set of rules:
  1. Names elements for any of the items in the device structure shall be alphanumeric. I.e., only upper and lower case alphanumeric characters (a-z, A-Z, 0-9) are allowed.
  2. Name elements for and discipline(Dis) shall be globally distinguishable.
  3. Name elements for device category (Cat) and type (Dev) shall be distinguishable under the same discipline.
  4. Names elements for any of the items in the device structure shall have 1 to 6 charachers.
